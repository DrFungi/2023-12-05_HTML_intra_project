Oracle Database (Oracle DB) est un système de gestion de base de données relationnelle (SGBDR) d'Oracle Corporation. Développée à l'origine en 1977 par Lawrence Ellison et d'autres développeurs, Oracle DB est l'un des moteurs de base de données relationnelles les plus fiables et les plus largement utilisés pour stocker, organiser et récupérer des données par type tout en maintenant des relations entre les différents types. Le système est construit autour d'un cadre de base de données relationnelle dans lequel les objets de données peuvent être directement accédés par les utilisateurs (ou une application frontale) par le biais d'un langage de requête structuré (SQL). Oracle est une architecture de base de données relationnelle entièrement évolutive et est souvent utilisée par des entreprises internationales qui gèrent et traitent des données sur des réseaux locaux et étendus. La base de données Oracle possède son propre composant réseau pour permettre les communications entre les réseaux. La base de données Oracle est également connue sous le nom de SGBDR Oracle et, parfois, simplement sous le nom d'Oracle.

Microsoft SQL Server est un système de gestion de base de données (SGBD) en langage SQL incorporant entre autres un SGBDR (SGBD relationnel ») développé et commercialisé par la société Microsoft. Il fonctionne sous les OS Windows et Linux (depuis mars 2016), mais il est possible de le lancer sur Mac OS via Docker, car il en existe une version en téléchargement sur le site de Microsoft.

MongoDB is a source-available cross-platform document-oriented database program. Classified as a NoSQL database program, MongoDB uses JSON-like documents with optional schemas. MongoDB is developed by MongoDB Inc. and current versions are licensed under the Server Side Public License (SSPL). MongoDB is a member of the MACH Alliance.

Tableau

  1. Fonctionnalités
    • Modèle orienté document: MongoDB a été créé pour s'adapter à l'esprit productif et flexible d'un développeur. Une façon d'atteindre cet objectif est de le rendre orienté document. Le modèle de document permet de stocker des données sous forme de documents regroupés en collections. Les documents sont autonomes, ce qui facilite la concentration des développeurs sur un ensemble de données particulier sans les diviser en tables. MongoDB utilise également le format BSON pour stocker les documents. BSON est un format JSON encodé en binaire qui permet de stocker des images, des vidéos, du texte, etc. Il est facile d'interagir avec MongoDB en utilisant un pilote spécifique au langage de programmation utilisé.
    • Réplication: Avec la réplication MongoDB, vous pouvez atténuer les risques de défaillance des données en répliquant les données sur plusieurs serveurs. Ces serveurs sont utilisés pour mettre à l'échelle la base de données, récupérer les données et servir de sauvegarde. Grâce à la réplication, il est également possible que le moteur de stockage de la base de données répartisse la charge de lecture sur les différentes répliques.
    • Équilibrage de charge: Un équilibrage de charge adéquat est fondamental pour la gestion de base de données afin d'assurer l'expansion dans les organisations à grande échelle. Avec des milliers voire des millions de flux et de requêtes clients, ils doivent être correctement répartis entre les différents serveurs pour maximiser les performances et réduire la congestion excessive. MongoDB gère efficacement les requêtes de lecture et d'écriture, équilibrant la charge entrante entre ses multiples serveurs et assurant la cohérence des données. Cela signifie qu'avec MongoDB, il n'est pas nécessaire d'utiliser des équilibreurs de charge supplémentaires.
  2. Forces
    • Haute Vitesse/Performances : La nature orientée document de MongoDB lui permet d'offrir une vitesse supérieure aux bases de données relationnelles traditionnelles. MongoDB tire parti du cache et de la mémoire vive (RAM), ce qui permet une récupération plus rapide et de meilleures performances. Cependant, cela signifie que vous avez besoin d'un système avec une quantité de RAM décente.
    • Prise en charge des requêtes Ad-hoc : MongoDB prend en charge les requêtes ad-hoc et a optimisé la manière dont ce type de requête est géré. Cela permet des mises à jour en temps réel des requêtes ad-hoc, contrairement aux requêtes de routine.
    • Flexibilité : MongoDB est un système de base de données NoSQL, ce qui le rend adapté à la fois aux données structurées et non structurées. Il est également sans schéma, ce qui signifie que vous pouvez stocker n'importe quel type de données sans être restreint par un schéma.
  3. Limites
    • Jointures complexes : MongoDB ne prend pas en charge les jointures comme le ferait une base de données relationnelle, ce qui rend difficile l'association de vos documents stockés. Vous pouvez le réaliser manuellement via du code, mais cela prend du temps et peut affecter les performances.
    • Grande dépendance aux bons index : Si vos index présentent des incohérences ou des erreurs, cela peut avoir un impact négatif sur la vitesse et les performances de MongoDB. Par conséquent, vous devrez passer du temps à corriger et à analyser les index que vous utilisez.
    • Données en double : MongoDB peut souffrir de données en double, ce qui rend difficile la gestion efficace de vos données.

PostgreSQL est un système de gestion de base de données relationnelle et objet (SGBDRO). C'est un outil libre disponible selon les termes d'une licence de type BSD.

Ce système est comparable à d'autres systèmes de gestion de base de données, qu'ils soient libres (comme MariaDB et Firebird), ou propriétaires (comme Oracle, MySQL, Sybase, DB2, Informix et Microsoft SQL Server). Comme les projets libres Apache et Linux, PostgreSQL n'est pas contrôlé par une seule entreprise, mais est fondé sur une communauté mondiale de développeurs et d'entreprises.

MySQL est un système de gestion de base de données relationnelle open source, soutenu par Oracle (RDBMS), basé sur le langage de requête structuré (SQL). Il fonctionne sur pratiquement toutes les plateformes, y compris Linux, UNIX et Windows. Bien qu'il puisse être utilisé dans un large éventail d'applications, MySQL est le plus souvent associé aux applications Web et à la publication en ligne.

MySQL est un composant important d'une pile d'entreprise open source appelée LAMP. LAMP est une plateforme de développement web qui utilise Linux comme système d'exploitation, Apache comme serveur web, MySQL comme système de gestion de base de données relationnelle et PHP comme langage de script orienté objet. (Parfois, Perl ou Python est utilisé à la place de PHP.)

Initialement conçu par la société suédoise MySQL AB, MySQL a été acquis par Sun Microsystems en 2008, puis par Oracle lorsqu'il a racheté Sun en 2010. Les développeurs peuvent utiliser MySQL sous la licence publique générale GNU (GPL), mais les entreprises doivent obtenir une licence commerciale auprès d'Oracle.

  1. Fonctionnalités
    • MySQL est open-source, ce qui signifie que ce logiciel peut être téléchargé, utilisé et modifié par n'importe qui. Il est gratuit et facile à comprendre. Le code source de MySQL peut être étudié et modifié en fonction des besoins. Il utilise la licence GPL, c'est-à-dire la licence publique générale GNU, qui définit les règles et réglementations concernant ce qui peut ou ne peut pas être fait avec l'application
    • MySQL stocke efficacement les données en mémoire pour garantir la cohérence des données et éviter les redondances. Par conséquent, l'accès et la manipulation des données avec MySQL sont rapides
    • Il contient plusieurs types de données tels que les entiers non signés, les entiers signés, les nombres à virgule flottante (FLOAT), les nombres à virgule double (DOUBLE), les caractères (CHAR), les caractères variables (VARCHAR), le texte, le blob, la date, l'heure, la date et l'heure, le timestamp, l'année, et ainsi de suite.
  2. Forces
    • Facile à apprendre – MySQL est relativement simple à apprendre et à utiliser, ce qui en fait un choix populaire pour les développeurs de tous niveaux. Il utilise un langage de requête simple et intuitif, ce qui facilite la récupération et la manipulation des données
    • Compatibilité étendue – MySQL est compatible avec une large gamme de plateformes, notamment Windows, Linux et macOS. Cela signifie qu'il peut être utilisé sur divers appareils, des ordinateurs de bureau aux appareils mobiles.
    • Hautes performances – MySQL est conçu pour des performances élevées, ce qui lui permet de gérer de grandes quantités de données et de trafic sans ralentissement. Il utilise des techniques d'indexation avancées et des mécanismes de mise en cache pour optimiser les performances, en en faisant un choix idéal pour les applications nécessitant un traitement de données en temps réel
  3. Limites
    • Scalabilité MySQL peut être difficile à mettre à l'échelle, notamment lorsqu'il s'agit de gérer de grandes quantités de données. Cela est dû à son architecture centralisée, qui peut entraîner des problèmes de performance à mesure que la base de données augmente en taille
    • Complexité MySQL peut être complexe à configurer et à entretenir, nécessitant des connaissances et des compétences spécialisées. Cela peut rendre son utilisation efficace difficile pour les petites organisations ou les particuliers
    • Sécurité MySQL peut être vulnérable aux violations de sécurité, notamment s'il n'est pas configuré correctement ou si les mises à jour ne sont pas appliquées en temps voulu. Cela peut entraîner la perte de données sensibles ou d'autres problèmes de sécurité
